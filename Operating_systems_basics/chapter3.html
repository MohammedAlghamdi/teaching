Creating Soft Link to a File

  • Command: ln -s /etc/hostname name
  • Explanation: This command creates a symbolic (soft) link named name that points to /etc/hostname. Soft links are pointers to the original file, allowing you to access the file from another location.
  • Example: ln -s /etc/hosts link_to_hosts

Creating Hard Links

  • Command: echo "Hello World!" > hello.txt followed by ln hello.txt bye.txt
  • Explanation: This set of commands first creates a file named hello.txt containing the text "Hello World!" and then creates a hard link named bye.txt pointing to the same physical location on the disk as hello.txt. Unlike soft links, hard links are indistinguishable from the original file.
  • Example: ln /etc/hosts copy_of_hosts

Extracting a Tar File

  • Command: For gzip compressed files: tar -xzvf files.tar.gz, for bzip2 compressed files: tar -xjvf files.tar.bz2
  • Explanation: These commands extract files from tar archives. The -x option specifies extraction, -z and -j specify the compression method (gzip and bzip2, respectively), -v enables verbose mode, and -f specifies the filename of the archive.

Creating a Tar File

  • Command: tar -czvf files.tar.gz hello.c bye.txt
  • Explanation: This command creates a compressed tar archive named files.tar.gz containing the files hello.c and bye.txt. The -c option specifies creation of an archive, -z enables gzip compression, -v enables verbose mode, and -f specifies the filename of the archive.

Vim Editor Commands

  • Commands and Explanations:
    • Exit vim: :q - Quits Vim. If there are unsaved changes, Vim will not allow you to quit.
    • Open a new or existing file: vim filename - Opens filename for editing in Vim. If filename does not exist, Vim starts with a new buffer.
    • Save a file in vim: :w - Saves the changes made to the open file.
    • Save and quit vim: :wq or :x - Saves changes and exits Vim.
    • Quit without saving: :q! - Quits Vim without saving changes.

Becoming Root User

  • Command: su -
  • Explanation: This command switches the current user to the root user, granting administrative privileges. The - option provides an environment similar to that the root user would expect.

Using Sudo Command

  • Commands: sudo less /var/log/secure and sudo !!
  • Explanation: The sudo command temporarily elevates privileges allowing a permitted user to execute a command as the root user. The sudo !! command repeats the last command entered in the terminal with sudo.

Environment Variables

  • Commands: View all environment variables: printenv, Set up environment variable values: export NAME="Kushal Das"
  • Explanation: Environment variables are dynamic-named values that affect the behavior of running processes on a computer. printenv displays all environment variables, while export sets a new environment variable or modifies an existing one.

Locate Command

  • Commands: locate firewalld and sudo updatedb
  • Explanation: The locate command searches for files by name quickly, using a database updated by updatedb. It's useful for finding files when you know part of the name.
