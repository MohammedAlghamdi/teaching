Overview

This chapter covers the basic commands and concepts related to file permissions in Linux, including the management of permissions, understanding of the PATH variable, and configuration files.

Commands and Concepts

  • ls command: Used to list files and directories along with their permissions.

    • Example: ls -l
    • Explanation: This command lists the contents of a directory in long format, showing file permissions, number of links, owner, group, size, and modification date.

  • chmod command: Changes the file mode bits to alter the access permissions of files and directories.

    • Example: chmod 755 myfile.txt
    • Explanation: This command changes the permissions of myfile.txt to read, write, and execute for the owner, and read and execute for the group and others.

  • PATH variable: A shell variable that specifies the directories in which the shell looks for commands.

    • Example: echo $PATH
    • Explanation: Displays the current directories in the PATH variable where the shell looks for executables.

  • ~/.bash_profile file: A configuration file for bash for users who can log in. It can set environment variables, update $PATH, and execute commands after login.

    • Example: Adding export PATH=$PATH:/new/directory to ~/.bash_profile
    • Explanation: This adds /new/directory to the PATH variable, extending the search path for executable files.

  • .bashrc file: A special configuration file for your bash terminal, used by users who cannot log in via standard methods. It allows you to define, delete, or update environment variables and commands for new interactive shells.

    • Example: Adding alias ll='ls -l' to ~/.bashrc
    • Explanation: Defines an alias ll for the command ls -l, making it easier to list files with detailed information.

  • /etc/profile file: Used to configure system-wide settings for all users when a new login shell is created.

    • Example: Adding PATH=$PATH:/usr/local/bin to /etc/profile
    • Explanation: Extends the PATH variable for all users, adding /usr/local/bin to their search path for executables.

  • which command: Finds the exact path of the executable being used by a command in the shell.

    • Example: which ls
    • Explanation: Shows the full path of the ls command executable.

  • Using which command to demonstrate $PATH variable functionality: Shows how the shell searches for commands in the directories specified in $PATH.

    • Example: which nonexistentcommand
    • Explanation: Attempts to find the executable for a command that doesn't exist, demonstrating how which searches the PATH.

  • she-bang or sha-bang in executable files: Indicates the interpreter path for executing scripts.

    • Example: Adding #!/bin/bash at the top of a script file.
    • Explanation: Specifies that the script should be executed using the Bash shell interpreter.
